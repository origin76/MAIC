# 第四章 实验结果与分析（草稿）

## 4.1 实验目标与研究问题

本章旨在围绕“基于信息交互的多智能体强化学习算法”这一主题，对本文所采用的基线算法、通信机制设计以及改进方法的有效性进行系统实验分析。结合前文方法设计，本章重点回答以下几个问题：

1. 在当前代码框架与实验设置下，为什么需要首先构建一个可靠的 MAPPO 基线，而不是直接在不稳定策略上叠加通信模块。
2. 在原型协作环境 `join1` 中，通信机制能否带来显著增益，以及其失效是否具有系统性特征。
3. 在更具代表性的 SMAC `5m_vs_6m` 任务中，vanilla MAPPO 是否能够形成足够强的 Backbone，从而为后续通信改进提供可信基础。
4. 在固定强基线的前提下，不同类型的轻量通信结构（v1-v5）分别解决了什么问题，其性能变化和机制变化体现了怎样的信息交互规律。

为保证结果分析的可解释性，本文在实验中主要采用以下三类统计量：

- 峰值胜率（`peak`）：测试胜率曲线中的最大值，用于衡量方法的上限能力。
- 最终胜率（`final`）：最后一个有效测试点的胜率，用于衡量训练结束时的结果。
- 末 5 点平均胜率（`last5`）：最后 5 个测试点的平均值，用于衡量方法后期稳定性。

其中，最终结果统计优先以 `results/sacred/*/cout.txt` 中记录的测试指标为准；绘图数据则优先采用导出的 TensorBoard JSON 文件，以保证曲线展示的完整性。

## 4.2 基线算法选择：为何以 MAPPO 作为主干

### 4.2.1 研究任务对基线稳定性的要求

本文的研究目标并非单纯证明“通信有用”，而是要进一步讨论：多智能体在集中式训练、分布式执行（CTDE）范式下，如何学习有效的信息编码方式、如何选择通信对象，以及如何将接收到的信息融合进局部决策。换言之，通信模块在本研究中是待分析与待改进的对象，而非预设正确的前提。

在这种研究目标下，如果基础策略网络本身不稳定，则后续任何通信改进都难以解释。具体而言，当实验结果不理想时，很难区分问题究竟来自：

- PPO 优化本身不稳定；
- centralized critic 设计不足；
- Backbone 表征能力不够；
- 还是通信模块本身的编码、路由与融合方式存在缺陷。

因此，在通信设计之前，首先构建一个足够稳定且具有较强性能上限的 MAPPO 基线，是本文实验主线能够成立的必要条件。

### 4.2.2 MAPPO 作为主干方法的适用性

本文选择 MAPPO 作为主干方法，主要出于以下考虑。

首先，MAPPO 与本文所强调的 CTDE 范式天然一致。Actor 在执行阶段仅依赖局部观测，符合去中心化执行要求；Critic 在训练阶段能够利用更充分的集中信息，从而提升价值估计质量，这与多智能体协作任务中的训练-执行分工高度契合。

其次，MAPPO 已成为多智能体强化学习中具有代表性的 on-policy 基线之一。选择 MAPPO 作为主线，不仅便于与公开工程实现和既有研究进行比较，也更有利于后续从“标准实现”逐步过渡到“带通信的改进实现”。

最后，从实验策略上看，PPO 的约束更新机制适合承载“微量通信注入”的设计思想。通信模块并不需要替代整个局部策略，而可以作为一个低带宽、低增益、残差式的修正分支，以较低风险地插入到已有 Backbone 中。

## 4.3 原型环境 `join1` 上的通信验证

### 4.3.1 选择 `join1` 的目的

尽管 `join1` 并非本文最终的主评测环境，但其在实验链条中具有重要作用。与 SMAC 相比，`join1` 的任务结构更简化，更适合早期观察通信协议是否能够带来基础性增益，以及不良通信结构是否会快速暴露出坍塌现象。因此，本文将 `join1` 视作通信机制的原型验证环境，而非最终结论环境。

### 4.3.2 `join1` 上的实验现象

已有实验结果表明，`join1` 上通信机制的有效性并非必然成立，而是高度依赖通信协议本身的约束形式。

在该环境中，`qmix` 能够学习到一定程度的协作行为，其代表性 run 在测试胜率上达到 `0.7400` 的峰值，最终胜率为 `0.5960`，末 5 点平均为 `0.6704`。相比之下，`budgeted_sparse_mappo` 在 `join1` 上表现极为突出，其代表性 run 在峰值、最终点和末 5 点平均三项指标上均达到 `1.0000`。这一结果说明，在该类环境中，受预算约束的稀疏通信确实具有显著优势。

然而，若通信结构设计不合理，则性能并不会出现“略弱但仍可训练”的状态，而是可能直接坍塌。例如，`maic`、`maic_parallel` 以及 `maic_parallel_join1_tuned` 在代表性 run 中的测试胜率几乎始终维持在 `0.0000`。这说明通信模块本身并不是天然的协同增益来源，其编码方式、通信对象选择与融合策略若缺乏结构约束，完全可能破坏策略学习过程。

### 4.3.3 `join1` 对后续设计的启示

`join1` 上的实验结论为后续在 `5m_vs_6m` 上的设计提供了明确启发。首先，通信机制必须受到约束，尤其应避免“所有人都听一点、最后得到平均噪声”的设计。其次，低带宽、稀疏化、受预算限制的通信形式往往比高自由度、强耦合的通信更稳定。最后，通信结构不应被预设为“越复杂越好”，而应围绕可解释性和信息有效性进行设计。

因此，`join1` 在本文中的主要价值，不在于作为最终 benchmark 提供最重要的胜率数字，而在于作为“通信协议试验台”，帮助识别哪些通信思想值得保留，哪些通信结构需要在更复杂任务中被规避。

## 4.4 SMAC `5m_vs_6m` 上的 MAPPO 基线构建

### 4.4.1 从“能训练”到“可信 Backbone”

与 `join1` 相比，SMAC `5m_vs_6m` 更能体现部分可观测条件下的多智能体协作难点。在该任务中，智能体既需要依赖局部观测完成基础微操，又必须通过协作实现集火与站位配合。因此，该环境更适合作为本文通信改进的核心评测场景。

在实验初期，本文首先围绕 vanilla MAPPO 的实现进行了多轮对齐与修正，包括 centralized critic 的规范化实现、active mask 与 proper time-limit 的补全，以及 PPO 学习率衰减、KL early stop 等稳定化策略的引入。实验表明，若这些基础细节处理不充分，即使 MAPPO 方向整体正确，也可能出现显著的 seed 方差。

### 4.4.2 `officialish` 主线的观察

较早的 `officialish` 版本已经在实现风格上向公开 MAPPO 工程靠拢，但其性能仍存在明显 seed 敏感性。以 `vanilla_mappo_sc2_5m6m_agentwise_centralized_semistable_officialish_1p5m_lrdecay_klstop_relaxactor` 为例，seed 1 的代表性 run 峰值可达到 `0.6000`，最终胜率为 `0.5750`；而 seed 2 与 seed 3 的结果则明显偏弱。这说明当前实现方向基本正确，但仅依靠单一配置尚不足以形成稳定 Backbone。

### 4.4.3 Warm-start 控制实验的意义

在选取较优 checkpoint 并继续进行 300k warm-start finetune 后，基线性能出现了明显改善。三组代表性 run 的峰值分别达到 `0.7125`、`0.7000` 与 `0.6125`，最终胜率分别达到 `0.6500`、`0.6500` 与 `0.5750`，末 5 点平均也维持在较高水平。

这一结果具有关键意义。它说明：

- 当前代码框架中的 vanilla MAPPO 并非“根本跑不起来”；
- 一旦训练过程稳定，MAPPO Backbone 本身具备较强的微操能力；
- 后续通信模块的比较可以建立在已有强策略之上，而不必反复从零训练一个不稳定的主干。

因此，本文后续的 v1-v5 通信实验均以此类强 Backbone 为基础展开，保证不同通信结构之间的比较更具有解释性。

## 4.5 基于强 Backbone 的轻量通信结构演化

在建立强 MAPPO Backbone 后，本文不再采用“大幅修改主干网络”的方式引入通信，而是遵循“低带宽、低增益、零初始化、残差注入”的设计原则，逐步探索不同通信结构。为便于叙述，本文将通信模块的发展划分为 v1 至 v5 五个阶段。

### 4.5.1 v1：最小侵入式通信适配器

v1 的主要目标是验证一个最基础的问题：在不改 learner、不改 critic、尽量不扰动 Backbone 的前提下，能否将一个极小的通信适配器挂接到 Actor 上，并保持策略仍然可训练。

实验结果表明，v1 已经能够表现出一定效果。`v1_detach` 的代表性 run 峰值达到 `0.6875`；`v1_end2end` 多个 seed 的峰值也大致维持在 `0.6250` 至 `0.6458` 区间。该阶段的主要结论是，轻量通信适配器可以在强 Backbone 上生效，但其后期收尾稳定性尚不理想，说明此时通信特征虽然能够被利用，但尚未形成足够稳定的语义结构。

### 4.5.2 v2：通信锐化与更积极的适配训练

v2 主要尝试解决 v1 中“通信虽然接入了策略，但注意力过于平均、适配速度偏慢”的问题。对应的改进方向包括更尖锐的注意力分布以及更积极的通信模块训练策略。

从结果看，v2 整体优于 v1 的平均水平。例如，`v2_sharp` 的代表性 run 在最终胜率和末 5 点平均上都表现较好；`v2_sharp_soft` 在多个 seed 下也能达到相对较高的峰值与较稳定的收尾。这说明，当通信注意力不再长期停留在“模糊平均”的状态时，轻量通信确实能得到更稳定的收益。

不过，v2 的进步仍然有限，其根本原因在于通信内容本身仍较为抽象。也就是说，仅仅优化“如何听”，尚不足以从根本上解决“传来的信息是否有明确物理意义”的问题。

### 4.5.3 v3：动作意图共享

v3 是当前通信主线中的一次关键跃迁。与 v1、v2 更偏向传递抽象 hidden feature 不同，v3 开始直接传递具有明确语义的动作意图，即“当前更倾向攻击哪个目标”这一信息。

实验结果表明，v3 在纯性能上达到了当前通信线中的第一梯队。`v3_end2end` 的代表性 run 峰值达到 `0.7000`；`v3_gatefloor` 更是将峰值进一步提高到 `0.7250`。这说明，当通信内容具有清晰的物理含义时，Actor 更容易学习到“何时听取队友信息、以及如何利用该信息”的策略。

从机制角度看，v3 的重要发现是：对于星际微操这类任务而言，动作意图是一类高价值、低歧义的消息载体。相比抽象的隐状态向量，动作意图更容易在稀疏奖励条件下形成可用语义，因此能够更直接地促进集火协同。

### 4.5.4 v4：面向攻击子空间的 Targeted Fusion

尽管 v3 在峰值表现上最为突出，但其强融合形式仍存在一定 seed 波动。为提升可解释性，v4 进一步收缩了通信模块的作用范围，不再试图修改整个 Actor 隐状态，而只在攻击 logits 子空间中引入小幅残差修正。

这种设计使通信模块的功能变得更清晰：通信不是在“替代本地策略”，而是在“本地攻击偏好”基础上提供一个有控制的协同偏置。实验结果显示，`v4_base` 虽然不是当前峰值最高的方法，但其代表性 run 在最终胜率和整体稳定性上表现较为均衡；而 `v4_softuse` 虽可进一步提高峰值，但会重新带来明显的跨 seed 波动。

因此，v4 的核心价值不完全在于绝对数值，而在于其良好的结构可解释性。就论文主方法而言，v4 是目前最适合做主图展示和机制阐述的一版。

### 4.5.5 v5：攻击流与移动流的双流解耦

在 v4 中，通信已被证明能够在攻击子空间中稳定发挥作用。由此，本文进一步提出一个更细化的问题：攻击协同与移动协同是否应由同一类通信消息来承担。基于这一考虑，v5 将通信结构进一步划分为攻击流与移动流两部分，尝试分别处理“打谁”和“怎么走”两类协同信息。

实验结果表明，朴素双流并不会自动优于单流结构。`v5_base` 的性能整体与 v4 相近，而移动流在没有强约束时，容易退化为高熵平均，导致传递的局部拓扑信息被均值化。为此，本文进一步引入 `top1move` 的硬稀疏设计，将移动流的通信对象限制为最重要的一个邻居。结果显示，`v5_top1move` 的表现较 `v5_base` 有明显改善，说明移动协同信息的关键不在于“是否增加额外流”，而在于“是否能够避免路由发散”。

随后引入的 `softplus` 版本在跨 seed 稳定性上有所改善，但并未显著提高峰值上界。这表明，在当前阶段，移动流的主要瓶颈并非 gate 的激活形式，而仍然是通信内容和稀疏路由方式本身。

### 4.5.6 本节综合分析

综观 v1 至 v5 的演化，可以得到以下规律。

首先，通信模块必须尽量避免破坏 Backbone，因而低带宽、低增益、残差注入是合理的基本原则。其次，仅仅优化注意力分布并不足以显著提升协同能力，通信内容的语义清晰度更为重要。再次，动作意图是当前任务中最有效的一类消息载体，而移动协调由于更依赖局部拓扑与空间关系，需要更强的结构约束。最后，若以论文主方法的可解释性为优先目标，v4 是当前最合适的方案；若以峰值性能为优先目标，则 v3 更具竞争力；若以机制发现为目标，则 v5 对“移动协同为何困难”给出了最直接的证据。

## 4.6 综合讨论

结合以上实验，可以将本文当前阶段的总体认识概括如下。

第一，通信并不天然带来协同收益。`join1` 中 MAIC 系列的失败已经表明，若消息编码方式、通信对象选择和融合策略缺乏约束，通信模块甚至可能完全破坏学习过程。

第二，在 SMAC `5m_vs_6m` 任务中，先建立一个可靠的 vanilla MAPPO Backbone 是后续通信研究的前提。只有当基线本身具备稳定、可信的协作能力时，通信模块的收益与缺陷才具有可分析性。

第三，信息载体的语义清晰度比“公式层面的复杂性”更重要。动作意图之所以在 v3 中表现突出，并不在于其数学形式更复杂，而在于其物理含义明确、与集火协同直接相关。

第四，通信应优先服务于确实需要协同的动作子空间，而非试图接管整个局部策略。v4 通过将通信限制为对攻击 logits 的小幅修正，证明了“受控制的局部融合”比“大而全的全局拼接”更稳健，也更易解释。

第五，移动协同较攻击协同更难学习。这一难点并非简单来自模型容量不足，而更可能源于移动协同依赖局部拓扑、相对位置和即时威胁，其有效信息天然比攻击意图更稀疏、更局部。因此，v5 中硬稀疏路由对移动流的改善，具有较强的机制解释意义。

---

## 4.7 本章小结

本章首先说明了本文为何选择 MAPPO 作为通信研究的主干基线，并通过 `join1` 原型环境验证了通信机制既可能带来显著收益，也可能因协议设计不当而彻底失效。随后，在 SMAC `5m_vs_6m` 环境中，本文构建了较为可靠的 vanilla MAPPO Backbone，并在其基础上依次探索了 v1 至 v5 五类轻量通信结构。

实验结果表明，轻量、残差式、低带宽的通信适配器能够在强 Backbone 上稳定工作；具有明确物理语义的动作意图共享相较于抽象隐层特征具有更高的有效性；而将通信作用限制在攻击子空间能够显著增强方法的结构可解释性。进一步地，双流通信实验还揭示了移动协同信息相较攻击协同信息更容易发散，因而需要更强的硬稀疏约束。

综上，本文当前阶段已形成一条较为清晰的实验主线：以强 MAPPO Backbone 为基础，采用受约束、可解释、面向协作子空间的轻量通信模块进行改进。这一结论为后续章节中进一步总结算法设计思想、提炼改进方法的理论含义以及撰写最终论文结论部分奠定了实验基础。
